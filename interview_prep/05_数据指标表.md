## 七、硬件与部署

### 7.1 机器人平台
- **Kuavo 人形(双臂)**:主力平台,4322/4611 等多构型
- **Kuavo 轮臂**:MT-9 / MT-7(MT-9 网口断连,后迁 MT-7)
- **夹爪 / 灵巧手**:四指夹爪、47 夹爪(标准化)
- **V47-V49**:实验室机器编号

### 7.2 相机配置
- **ZED-M 双目** + FoundationStereo(8Hz 深度)
- **4 摄方案**:头部 + 胸部(低视角)+ 双肩(realsense / 奥比中光)
- **胸部低视角相机**:解决手-箱交互遮挡(0906 引入)

### 7.3 部署链路
- 训练:**H100 + Docker** 转数 → 多卡 accelerate
- 推理:.safetensors → ONNX → **TensorRT** → Orin
- Orin Jetpack 5.1.3
- **kuavo-manip 轻量化 SDK v0.1.0**(1128/1219):打包本地 SDK / IK / msgs

### 7.4 采数方式(三合一)
1. **原子技能**(脚本化,效率最高,一天 100+ 条)
2. **VR 遥操**(MetaVR / Pico / Quest3)
3. **键盘控制 IK 微调**(少量精调)
4. 行为树(bt)搬放辅助

**回答套路**:面试官问到 Groot 85% 时,正确话术是:

> "这个 85% 是团队 4 类箱体混训的压测结果,由杨小康主导 Groot 的训练框架。我在其中负责的部分是**数据侧的增强管线** —— 用 Cosmos-transfer 给 Groot 训练数据做背景和光线的 OOD 扩展,因为我们发现 Groot 在杂乱背景下成功率降到 83%、倾斜摆放降到 0%,所以数据增强是后续想把 OOD 继续拉起来的关键路径。"

这段话,既讲清了 85% 的团队归属,又把你的主导工作自然接出来,同时展现你对整个项目瓶颈的理解。

# 02 简历条目 —— 中英双语,3 个长度版本

> **用途**:直接粘贴到简历上。三个版本按简历空间选用:
> - **3 行版**:简历已经很满,给这个项目留的空间有限
> - **5 行版**(推荐):大多数实习生简历的标准长度
> - **8 行版**:项目是简历绝对主角,占半页
>
> **撰写原则**:
> 1. **动词优先**(主导 / 负责 / 参与 / 完成 / 搭建 / 对比 / 优化)
> 2. **数字优先**(85% / 30Hz / 2900+ / 6 小时 / 4 类箱体)
> 3. **模糊责任用 "团队"**,个人贡献用 "主导 / 参与"
> 4. **不堆砌名词**,每个技术词要么能解释、要么直接删
> 5. **不出现"我"这个字**(简历第一人称省略是惯例)

## 词条使用建议

### 当简历是**英文简历**(海外 / 外企)
- 用英文 Version B,这是招聘方最习惯的长度
- 技术名词不翻译(Groot / OpenVLA / ACT / LeRobot 保持英文)
- "Owned" 比 "Led" 在 IC / intern 语境下更自然("Led" 带团队感较强)

### 当简历是**中文简历**(国内大厂 / 国内机器人公司)
- 用中文 Version B
- 技术名词可以适度英文(Groot VLA / OpenVLA / Cosmos-transfer)
- "主导 / 参与 / 团队成果" 三类标签在面试时会被反复核验,**不要虚报**

### 当遇到**空间紧张**(比如简历要压 1 页)
- 用 Version A
- 保留 3 个核心数字:85% / 30Hz / 2900+
- 可以省略所有 OOD 细节,保留"Cosmos-transfer 数据增强"这个亮点词

### 当遇到**简历以项目为主角**(研究向求职、博士申请)
- 用 Version C
- 加一段"Key Insights / Findings"讲**个人反思** —— 机器人公司技术负责人级别的面试官会欣赏

## 面试官可能对简历的"挑错"预判

1. **"你说主导 Cosmos-transfer,具体做到哪一步?"**
   → 诚实讲:"启动到初步 pipeline,量化 AB 对比是下一阶段的工作。这也是我希望能继续做的方向。"

2. **"85% 是你做的吗?"**
   → 见 `03_STAR讲稿.md` 中的诚实话术。

3. **"你用过这么多 VLA 模型,随便讲一个说说架构?"**
   → 选你最熟的讲(推荐 ACT 或 Groot 的 SigLIP 部分),其他的只在被问到时讲。

4. **"为什么用 SAM3 不用 SAM 或 SAM2?"**
   → SAM2 视频分割能力强,SAM3 在开放词汇和细节上更好。**坦诚:这是基于文档和开源 benchmark 的选型,实测对比还没做完**。

5. **"LeRobot v3.0 和 v2 有什么改进?"**
   → Multi-Dataset 并行加载是最大改进,其他 API 层变化相对小。**如果不熟就说"主要用到 Multi-Dataset 能力"**。

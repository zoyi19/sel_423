## 二、模型架构(简版)

```
multi-cam images   language prompt    state(可选,π0)
       │                  │                 │
       ▼                  ▼                 │
   ┌──────┐          ┌──────────┐           │
   │SigLIP│          │ Tokenizer│           │
   └──┬───┘          └────┬─────┘           │
      │                   │                 │
      └───────────┬───────┘                 │
                  ▼                         │
       ┌──────────────────┐                 │
       │  PaliGemma-3B    │ ◄───────────────┘
       │  (Gemma-2.6B)    │
       └────────┬─────────┘
                │ KV pool 共享
                ▼
       ┌────────────────────┐
       │  Action Expert     │ ◄── adaRMS time(π0.5)
       │  (Gemma-300M)      │
       └────────┬───────────┘
                ▼
       Flow Matching head → action chunk (B, H=50, action_dim)
       推理 10 步 Euler ODE,~50ms / chunk
```

**两个核心组件**:
- **PaliGemma-3B** = SigLIP-So400m/14(0.4B 视觉)+ Gemma-2.6B(LLM)
- **Action Expert** = Gemma-300M 变体(width 1024,18 层),**和 PaliGemma 共享 attention pool**(不是 cross-attention)

**关键约束**:两个 expert 的 `head_dim=256, num_heads=8, num_kv_heads=1` 必须严格相等(共享 attention 的硬约束)。

## 三、三大关键技术点

### 3.1 Flow Matching action head(vs Diffusion)

- 学**速度场** v(x_t, t),不是噪声
- 训练:`x_t = t·noise + (1-t)·action`(直线插值),`u_t = noise - action`(常数!)
- Loss:`MSE(v_θ(x_t, t), u_t)`
- 推理:Euler ODE,**10 步够**(vs DDPM 50-100 步)

### 3.2 adaRMS time conditioning(π0.5 vs π0 唯一架构差异)

- π0:time emb 拼接到 action token,过 MLP
- π0.5:**每层 RMSNorm 的 scale/shift/gate 由 time 动态生成**(借鉴 DiT 的 adaLN-Zero)
- **零初始化** modulation:训练初期等价普通 RMSNorm,**不破坏 PaliGemma 预训练权重**

### 3.3 Heterogeneous Co-training

同一 batch 里混合多种监督任务:
- 30% IL(action prediction)
- 30% Detection(bounding box)
- 20% Captioning
- 20% Semantic subtask

→ 防止 VLM 在 IL 训练中**遗忘 grounding 能力**(catastrophic forgetting)。

### 4.1 模型初始化(`__init__`)

```python
class Pi0(_model.BaseModel):
    def __init__(self, config: pi0_config.Pi0Config, rngs: nnx.Rngs):
        super().__init__(config.action_dim, config.action_horizon, config.max_token_len)
        self.pi05 = config.pi05
        paligemma_config = _gemma.get_config(config.paligemma_variant)         # gemma_2b
        action_expert_config = _gemma.get_config(config.action_expert_variant) # gemma_300m

        # 用 nnx_bridge 桥接 jax flax(linen)和 nnx,Gemma/SigLIP 是 linen 风格
        llm = nnx_bridge.ToNNX(
            _gemma.Module(
                configs=[paligemma_config, action_expert_config],   # 两 expert 一起
                embed_dtype=config.dtype,
                adarms=config.pi05,                                  # 关键:π0.5 才用 adaRMS
            )
        )
        # use_adarms=[False, True]:PaliGemma 普通 RMSNorm,Action Expert 用 adaRMS
        llm.lazy_init(rngs=rngs, method="init",
                      use_adarms=[False, True] if config.pi05 else [False, False])

        img = nnx_bridge.ToNNX(
            _siglip.Module(num_classes=paligemma_config.width,
                           variant="So400m/14", pool_type="none", scan=True,
                           dtype_mm=config.dtype)
        )
        img.lazy_init(next(iter(config.fake_obs().images.values())), train=False, rngs=rngs)
        self.PaliGemma = nnx.Dict(llm=llm, img=img)

        self.action_in_proj = nnx.Linear(config.action_dim, action_expert_config.width, rngs=rngs)

        if config.pi05:
            # π0.5: time emb 单独 MLP,作为 adaRMS 条件
            self.time_mlp_in  = nnx.Linear(action_expert_config.width, action_expert_config.width, rngs=rngs)
            self.time_mlp_out = nnx.Linear(action_expert_config.width, action_expert_config.width, rngs=rngs)
        else:
            # π0: state 单独投影 + action+time concat 后过 MLP
            self.state_proj = nnx.Linear(config.action_dim, action_expert_config.width, rngs=rngs)
            self.action_time_mlp_in  = nnx.Linear(2 * action_expert_config.width, action_expert_config.width, rngs=rngs)
            self.action_time_mlp_out = nnx.Linear(action_expert_config.width, action_expert_config.width, rngs=rngs)

        self.action_out_proj = nnx.Linear(action_expert_config.width, config.action_dim, rngs=rngs)
```

**讲述要点**:
1. `super().__init__()` 必须先调,初始化 nnx.Module 内部状态
2. `nnx_bridge.ToNNX`:把 linen 风格的 Gemma/SigLIP 包装成 NNX(过渡兼容)
3. `lazy_init`:JAX 风格延迟初始化,根据输入 shape 推断参数
4. `pi05` 标志位控制三处:adaRMS / 去 state token / time emb 处理方式

### 4.2 Flow Matching 训练 loss(`compute_loss`)

```python
def compute_loss(self, rng, observation, actions, *, train=False):
    preprocess_rng, noise_rng, time_rng = jax.random.split(rng, 3)
    observation = _model.preprocess_observation(preprocess_rng, observation, train=train)

    batch_shape = actions.shape[:-2]
    noise = jax.random.normal(noise_rng, actions.shape)

    # 关键 trick:time 用 Beta(1.5, 1) 偏向小 t(更接近目标 action)
    # 让模型在"接近最终答案"的时刻学得更精细
    time = jax.random.beta(time_rng, 1.5, 1, batch_shape) * 0.999 + 0.001
    time_expanded = time[..., None, None]

    # 直线插值 + 真实速度场(常数!)
    x_t = time_expanded * noise + (1 - time_expanded) * actions
    u_t = noise - actions

    # prefix(image+text) 和 suffix(state+action+time)一起 forward
    prefix_tokens, prefix_mask, prefix_ar_mask = self.embed_prefix(observation)
    suffix_tokens, suffix_mask, suffix_ar_mask, adarms_cond = self.embed_suffix(observation, x_t, time)
    input_mask = jnp.concatenate([prefix_mask, suffix_mask], axis=1)
    ar_mask = jnp.concatenate([prefix_ar_mask, suffix_ar_mask], axis=0)
    attn_mask = make_attn_mask(input_mask, ar_mask)
    positions = jnp.cumsum(input_mask, axis=1) - 1

    (prefix_out, suffix_out), _ = self.PaliGemma.llm(
        [prefix_tokens, suffix_tokens], mask=attn_mask, positions=positions,
        adarms_cond=[None, adarms_cond],   # π0.5 才有 adarms_cond
    )
    v_t = self.action_out_proj(suffix_out[:, -self.action_horizon:])

    return jnp.mean(jnp.square(v_t - u_t), axis=-1)   # MSE on velocity field
```

**讲述要点**:
1. **time 用 Beta(1.5, 1)** 不是 uniform → 偏向"接近 target"的时刻
2. **直线插值**:`x_t = t·noise + (1-t)·action`
3. **真实速度场是常数** `u_t = noise - action`(因为路径直线)
4. **prefix-LM mask**:image/text 之间双向 attend,action 因果

### 4.3 推理 sample(KV cache + Euler ODE)

```python
def sample_actions(self, rng, observation, *, num_steps=10, noise=None):
    observation = _model.preprocess_observation(None, observation, train=False)
    dt = -1.0 / num_steps                              # 注意!t 从 1 走到 0,所以 dt 负
    batch_size = observation.state.shape[0]
    if noise is None:
        noise = jax.random.normal(rng, (batch_size, self.action_horizon, self.action_dim))

    # === 关键加速:prefix 只 forward 一次,缓存 KV ===
    prefix_tokens, prefix_mask, prefix_ar_mask = self.embed_prefix(observation)
    prefix_attn_mask = make_attn_mask(prefix_mask, prefix_ar_mask)
    positions = jnp.cumsum(prefix_mask, axis=1) - 1
    _, kv_cache = self.PaliGemma.llm([prefix_tokens, None],
                                      mask=prefix_attn_mask, positions=positions)

    # === Euler ODE 步进:每步只 forward suffix,复用 prefix KV ===
    def step(carry):
        x_t, time = carry
        suffix_tokens, suffix_mask, suffix_ar_mask, adarms_cond = self.embed_suffix(
            observation, x_t, jnp.broadcast_to(time, batch_size)
        )
        suffix_attn_mask = make_attn_mask(suffix_mask, suffix_ar_mask)
        prefix_attn_mask = einops.repeat(prefix_mask, "b p -> b s p", s=suffix_tokens.shape[1])
        full_attn_mask = jnp.concatenate([prefix_attn_mask, suffix_attn_mask], axis=-1)
        positions = jnp.sum(prefix_mask, axis=-1)[:, None] + jnp.cumsum(suffix_mask, axis=-1) - 1

        (prefix_out, suffix_out), _ = self.PaliGemma.llm(
            [None, suffix_tokens],            # prefix 不再 forward
            mask=full_attn_mask, positions=positions,
            kv_cache=kv_cache,                # 复用 prefix KV
            adarms_cond=[None, adarms_cond],
        )
        v_t = self.action_out_proj(suffix_out[:, -self.action_horizon:])
        return x_t + dt * v_t, time + dt      # Euler step

    def cond(carry):
        x_t, time = carry
        return time >= -dt / 2

    x_0, _ = jax.lax.while_loop(cond, step, (noise, 1.0))
    return x_0
```

**讲述要点**:
1. **prefix KV cache 是核心加速**:VLM forward 只跑 1 次,suffix 跑 10 步复用,**省 ~10×**
2. **dt 是负数** = -1/N(t 从 1 走到 0,代码注释里作者承认了符号反过来)
3. JAX `while_loop` JIT 编译展开,GPU 利用率高

### 4.4 Prefix-LM Attention Mask(实现"prefix 双向 + suffix 因果")

```python
def make_attn_mask(input_mask, mask_ar):
    """
    [0 0 0 1 1 1] → prefix-LM(前 3 个互相 attend,后 3 个 causal)
    """
    mask_ar = jnp.broadcast_to(mask_ar, input_mask.shape)
    cumsum = jnp.cumsum(mask_ar, axis=1)
    # token i 能 attend token j 当且仅当 cumsum[i] >= cumsum[j]
    attn_mask = cumsum[:, None, :] <= cumsum[:, :, None]
    valid_mask = input_mask[:, None, :] * input_mask[:, :, None]
    return jnp.logical_and(attn_mask, valid_mask)
```

> image+text 之间**双向 attend**,action token 对前面**因果**。

## 五、π 系列进化路线(详解)

### π0(2024.10)

- 基础架构:PaliGemma + Action Expert 双 expert 共享 attention pool
- Flow Matching action head(直线 ODE 10 步)
- 训练:OXE + 自采真机数据

### π0-fast(2024.12)

- **FAST tokenization**:把连续 action 离散成"频域 token",改用自回归生成
- 推理快(token AR 比 ODE 快),但精度略降
- **路线分叉**:之后主线还是回到 Flow Matching(π0.5)

### π0.5(2025.04,arxiv 2504.16054)

- **adaRMS** time conditioning(每层感知 time)
- **去 state token**(可能融到 prompt)
- **Heterogeneous co-training**(IL + detection + caption + subtask 多任务混训)
- **首次实现**:陌生家庭做家务(扫地 / 整理床)

### π*0.6(2025.11,arxiv 2511.14759)

**RECAP** = RL with Experience and Corrections via Advantage-conditioned Policies

**三阶段训练**:
1. Offline RL pretrain(advantage-conditioned policy)
2. Demos fine-tune(IL)
3. **On-robot RL with corrections + reward**

**核心 trick:Advantage-conditioned policy**
- 训练时 condition 真实 advantage
- 推理时永远 condition 'good'(高 advantage 标签)→ 自然倾向高 reward 行为
- **绕开真机 PPO 难题**:no on-policy / no reward hacking / no sample efficiency

**战绩**:hard tasks **2x throughput**,~50% fewer failures,**13 小时连续部署**(叠衣 / 咖啡机)。

> 标语:**"RL is Back"**(PI 把 RL 重新带回 robotics)

### π0.7(2026.Q1)

- 主打 **Compositional Generalization**(任务组合泛化)
- 见过原子任务("拿杯子"+"倒咖啡")可以组合做复合("拿杯子并倒咖啡")
- 长视地任务上更稳
